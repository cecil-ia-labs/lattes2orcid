# Senior Creative Content Write

Use this skill when the project needs better product-facing text, not more text.

## Goal

Turn robotic or institutional copy into language that feels:

- direct
- human
- specific
- useful
- easy to scan

## Priorities

1. Lead with the real benefit.
2. Cut generic phrasing and filler.
3. Prefer plain language over internal or technical jargon when the user is not choosing technical behavior.
4. Keep CTAs short and obvious.
5. Make footers, helper text, and empty states feel intentional instead of forgotten.

## What Good Copy Looks Like

- “reduz horas de trabalho manual”
- “envie o XML e baixe o `.bib`”
- “nenhum aviso por enquanto”

## What To Avoid

- vague mission statements
- inflated startup language
- repetitive claims about innovation or interoperability without a concrete payoff
- robotic phrases that read like generated filler

## Editing Guardrails

- Improve tone and clarity without changing product logic.
- Preserve factual accuracy.
- Prefer shorter sentences when the meaning stays intact.
- Keep technical precision where it matters, especially around privacy, limits, and file handling.
