# ORCID Source Docs Navigation

Use this skill to ground ORCID integration decisions in the official docs, not memory.

## What To Read First

- ORCID integration getting-started guide
- ORCID member API and record schema docs
- ORCID integration checklist
- Any repo-linked spreadsheet or checklist that defines acceptance criteria

## Workflow

1. Identify the exact ORCID question: OAuth, scopes, record write, ownership, or display rules.
2. Pull the official source of truth for that question.
3. Summarize only what affects the repo design or implementation.
4. Flag any mismatch between desired UX and ORCID constraints.

## Output Standard

- State the supported path.
- State the unsupported path or restriction.
- Call out required scopes, identifiers, and ownership rules.
- Prefer concise, implementation-ready notes over generic summaries.
